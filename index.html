如何用hexo打造一个静态博客

下载需要的工具

nodejs, git

通过node的包管理器npm下载cmpn

管理员运行cmd，输入下列命令安装cnpm
npm install -g cnpm –registry=https://registry.npm.taobao.org 先安装cnpn不然直接安装hexo非常慢，可以理解cnpm为vpn

输入下列命令通过cnpm安装hexo
cnpm install -g hexo-cli

为自己的博客创建文件夹blog并初始化hexo，命令: hexo init，两个地方可以输入，看安装情况，安装git到第三步: Adjusting your PATH environment时有三个选择
1.只能通过Git Bash工具或Git cmd访问git
2.可以通过第一种方法访问git也可以通过cmd访问
3.我也不懂，不管他就好
我选择的是一种，第二种反正系统变量乱了可以上网找，绝对有，而且一般来说这种基本的错误git不会有，根据自己的选择，输入命令，这里要提一下git bash的命令用起来不太习惯cd命令无法直接到其他盘，因为我blog文件夹存在于文档，而文档存在于C盘，这种情况要么上网找命令或者直接找到自己的blog文件右键可以看到 git gui(界面) git bash(终端)，本文只讲bash的用法

初始化好hexo就可以运行了，输入命令:hexo s
如果成功会出现一个本地地址:localhost:4000，一般不会失败在这一步

如果想要创建博客文章，通过命令: hexo new “blogName”
创建后的博客文章都会出现在blog/source/posts下(posts前面有一个下划线，在vim编辑器中输入下划线有点问题，因为是默认安装的，文本编辑贼难受)，如果想要编写效率和美观可以在安装git第二步的时候选择对应编辑器，根据自己习惯而定
cd目录到该目录中，输入命令: vim blogName.md(如果你安装的是别的编辑器，直接在外面打开就好了右键该博客文件打开方式更改为自己的编辑器)

进入到输入(只要文件是空的一进去就是输入模式否则是n编辑模式)，这个我目前也不是太懂，只知道支持markdown写法，其他的都不懂，编写完后ESC退出编辑模式，然后底部输入:wq(保存文件)
vim blogName.md 进入vim编辑器
vim编辑器有三种模式:
1.输入模式
切记不要把文件中的格式更改，我在这里搞了半个小时真tm恶心，文件格式如下
—
title: blogName
date: 0000-00-00 00:00:00
tags:
—
## 标题
内容
—
2.编辑模式
3.末行模式

如果是空文件，则进入就是输入模式，输入完成后按ESC键进入编辑模式，再按:键进入末行模式(冒号一定要英文，上面一样)，然后输入wq(保存并退出vim编辑器)
末行模式有很多命令，都在下面文章中最后一条中展示

退出vim编辑器后通过cd命令退到blog层: cd ../../(如果不知道当前处在哪一层可以使用pwd查看，也可以使用ls命令查看当前目录下文件)
然后输入hexo clean(可以理解为清理缓存)
接着输入: hexo g(重新生成，可以理解重新部署文件)
然后运行 hexo s就可以了
